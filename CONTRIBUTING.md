# Contributing

Thanks for your interest. A few rules:

1. Open an issue before a non-trivial PR.
2. Conventional commits: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`.
3. `task lint` and `task test` must pass locally.
4. License headers: every new Go / TS file needs an SPDX identifier. Run `task license:apply` to auto-apply.

Small PRs are merged fast. Large PRs should link to a discussion or issue.
